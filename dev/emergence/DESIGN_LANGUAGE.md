# DESIGN_LANGUAGE.md

## Project Emergence

### Version 0.1

> Design does not decorate thought.
>
> Design reveals thought.

---

# Purpose

This document defines the visual language of Project Emergence.

It does not describe an interface.

It defines how philosophical principles become perceptible through visual behavior.

Every visual decision must be derivable from FOUNDATION.md and consistent with VISUAL_SYSTEM.md.

If a visual element cannot be justified by a first principle, it should not exist.

---

# Design Philosophy

Project Emergence is not a writing application.

It is an environment where thought becomes visible.

The interface should feel quieter than the mind using it.

The system never competes with thought.

It supports it.

---

# First Principle

Structure precedes appearance.

Visual beauty is the consequence of structural coherence.

Decoration is never introduced independently.

---

# Visual Grammar

The visual language follows the structural grammar defined by Psicoandino.

Geometry carries meaning.

Meaning precedes aesthetics.

---

## Circle

Represents origin.

Continuity.

Identity.

Persistence.

Every Emergence is visually represented as a circle because it is the beginning of a lineage.

A circle is never used merely as decoration.

---

## Line

Represents lineage.

Direction.

Relationship.

Temporal continuity.

A line does not connect objects.

It reveals ancestry.

---

## Triangle

Represents intentional change.

Decision.

Transformation.

Tension.

Triangles appear only where meaningful divergence occurs.

---

## Square

Represents systems.

Containers.

Organization.

Squares organize thought.

They never represent thought itself.

---

## Negative Space

Negative space is active.

It represents possibility.

Silence.

Unknown relationships.

The interface should never attempt to occupy all available space.

Absence is meaningful.

---

# Motion

Motion communicates change of state.

Never excitement.

Animations should be nearly imperceptible.

Movement exists to communicate meaning.

Never delight alone.

---

## Expression 

appears naturally.

Emergence is revealed through Seal.

The transition should communicate permanence,

not creation.

---

## Seal

Seal is a ritual.

Not a save operation.

The transition should communicate permanence through subtlety.

Seal transforms an editable Expression into an immutable Emergence.

The animation communicates recognition,

not storage.

The writer should feel acknowledgement rather than completion.

---

## Descendant

A Descendant grows from an ancestor.

It should never appear disconnected.

The transition should communicate continuation rather than duplication.

---

## Publication

Publication is calm.

Publishing is not celebration.

It is simply a change in visibility.

No confetti.

No applause.

No reward mechanics.

---

# Visual Hierarchy

The hierarchy follows thought rather than interface conventions.

Priority is always:

1. Thought
2. Lineage
3. State
4. Controls
5. Decoration

The interface must never prioritize controls over content.

---

# Color

Color expresses structural roles.

It never carries the entire meaning.

Meaning must remain understandable in monochrome.

Color reinforces.

It never defines.

Project Emergence inherits its palette from VISUAL_SYSTEM.md.

Each color represents a frequency rather than a permanent hierarchy.

---

# Typography

Typography should disappear into reading.

The writer should remember the thought.

Not the font.

Titles may establish identity.

Body text should maximize clarity.

---

# Rhythm

Visual rhythm follows cognition.

Writing alternates between:

Expression.

Silence.

Recognition.

Reflection.

The interface should breathe accordingly.

---

# Density

Default density is low.

Thought requires space.

Crowding increases cognitive friction.

Information appears only when context makes it meaningful.

---

# Visibility

Not everything should always be visible.

Complexity should exist structurally.

Not visually.

The interface progressively reveals depth.

---

# Genealogy

Genealogy is rarely shown in full.

Only the lineage relevant to the current thought should remain visible.

The complete genealogy always exists.

It is simply not always rendered.

---

# Time

Time is not represented primarily through dates.

Time is represented through ancestry.

Chronology supports genealogy.

It never replaces it.

---

# States

Thoughts move through recognizable states.

These states communicate maturity.

Not value.

The current working vocabulary is:

🌱 Emergence

🌿 Reflection

🌳 Matured *(working term)* Reserved for future versions.

📖 Published

These symbols should remain visually restrained.

They function as quiet orientation rather than status badges.

---

# Interaction Visibility

Every interaction should answer one question:

"What changed?"

Never:

"What just happened?"

The interface explains itself through continuity.

---

# Silence

Silence is a design element.

Whitespace is not empty.

It provides cognitive breathing room.

Silence allows thought to become visible.

---

# Emotional Tone

The interface should evoke:

Calm.

Precision.

Curiosity.

Respect.

Never urgency.

Never productivity anxiety.

Never gamification.

Never competition.

---

# Design Test

Every visual decision must satisfy the following questions:

Does it reveal structure?

Does it reduce cognitive noise?

Does it preserve lineage?

Does it respect silence?

Would removing it make the philosophy weaker?

If the answer to the final question is "no," the element should not exist.

---

# Invariant

The interface exists to reveal the evolution of thought without becoming more visible than the thought itself.
